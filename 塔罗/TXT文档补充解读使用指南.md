# TXT 文档补充解读使用指南

## 功能说明

现在您可以通过编辑一个简单的 txt 文档来为所有 22 张塔罗牌添加补充解读。应用会在启动时自动加载这个文件，并在"普通解读"模式下展示补充内容。

---

## 文件位置

文件名：**数据库补充22张卡牌解读.txt**
位置：与 `index.html` 在同一文件夹中

---

## 文件格式说明

### 基本结构

```
【卡牌】[卡牌编号和名称]
【补充解读】
[这里是补充解读的内容，可以多行]
【来源】[来源标识，如：心理学派补充、实用主义派补充等]
```

### 完整示例

```
【卡牌】0. The Fool (愚者)
【补充解读】
在心理学角度看，愚者代表了人类的原始冲动和对自由的渴望。这张牌提醒我们，有时候真正的智慧来自于敢于放下理性的束缚，去追随内心的声音。
【来源】心理学派补充
```

## 如何编辑

### 第 1 步：打开文件
用任何文本编辑器打开 `数据库补充22张卡牌解读.txt`

推荐编辑器：
- Windows：记事本（Notepad）或 VS Code
- Mac：TextEdit 或 VS Code
- Linux：nano、vim 或任何文本编辑器

### 第 2 步：找到要编辑的卡牌

例如，要编辑"1. The Magician (魔术师)"的补充解读，找到：

```
【卡牌】1. The Magician (魔术师)
【补充解读】
[原有内容]
【来源】[原有来源]
```

### 第 3 步：修改补充解读内容

在 `【补充解读】` 和 `【来源】` 之间的部分就是补充内容。您可以：
- 替换为自己的解读文字
- 保留原有内容
- 添加更多段落（可以多行）

### 第 4 步：保存文件

保存为 **UTF-8 编码**（重要！中文字符需要 UTF-8）

在大多数编辑器中：
- **VS Code**：右下角选择 "UTF-8"
- **记事本**：另存为 → 编码选择 UTF-8
- **TextEdit**：格式 → 选择"纯文本"，再保存时选择 UTF-8

### 第 5 步：刷新应用

1. 在浏览器中打开塔罗应用
2. 点击"进入仪式"按钮
3. 应用会自动加载最新的补充解读

## 常见问题

### Q: 刷新后补充解读没有显示
**A:** 
1. 检查文件是否保存
2. 确认文件编码是 UTF-8
3. 检查浏览器控制台是否有错误（F12 → Console）
4. 清空浏览器缓存后再试

### Q: 可以删除某些卡牌的补充解读吗？
**A:** 可以。将该卡牌条目的补充内容留空即可：
```
【卡牌】0. The Fool (愚者)
【补充解读】

【来源】
```

### Q: 可以添加新的卡牌吗？
**A:** 当前系统只支持 22 张大牌。如果需要添加小阿卡纳（40 张牌），需要修改应用代码。

### Q: 文件损坏了怎么办？
**A:** 
1. 下载备份文件（提供）
2. 或重新编辑一个新的 txt 文件，按照格式重新填充内容

### Q: 可以使用其他语言吗？
**A:** 可以。系统支持任何 UTF-8 编码的语言。

## 最佳实践

### 写作建议
1. **简洁明了**：补充内容 200-400 字为宜
2. **多角度**：从不同派系（神秘学、心理学、实用等）提供解读
3. **实用性**：包含可应用于现实生活的建议
4. **一致性**：保持与原有解读的风格和语调

### 内容质量
- 避免重复原有解读内容
- 提供新的视角或深度
- 包含具体的例子或场景
- 保持积极且有建设性的态度

### 文件维护
- 定期检查格式是否正确
- 确保所有 22 张卡牌都有条目
- 使用一致的【来源】标签

## 故障排除

### 问题 1：浏览器显示 404 错误
**原因**：找不到 txt 文件
**解决**：
1. 确认文件与 index.html 在同一文件夹
2. 确认文件名完全匹配：`数据库补充22张卡牌解读.txt`
3. 确认没有多余的空格或特殊字符

### 问题 2：显示乱码
**原因**：文件编码不是 UTF-8
**解决**：
1. 用支持编码设置的编辑器（如 VS Code）打开
2. 选择"以 UTF-8 重新编码"
3. 保存

### 问题 3：某些卡牌的补充内容没有显示
**原因**：卡牌名称不匹配或格式错误
**解决**：
1. 检查 【卡牌】 后面的名称是否与应用中的完全一致
2. 检查是否有多余空格
3. 参考卡牌编号对照表

## 高级用法

### 自动化导入
如果有大量补充内容来自数据库或其他来源，可以写一个脚本自动生成 txt 文件：

```python
# Python 示例伪代码
cards = [...] # 从数据库读取卡牌列表
output = ""
for card in cards:
    output += f"【卡牌】{card.name}\n"
    output += f"【补充解读】\n{card.supplement}\n"
    output += f"【来源】{card.source}\n\n"
    output += "---卡牌分隔线---\n"
with open('数据库补充22张卡牌解读.txt', 'w', encoding='utf-8') as f:
    f.write(output)
```

### 批量更新
使用文本编辑器的"查找和替换"功能（Ctrl+H 或 Cmd+H）进行批量更新：
- 替换某个派系的所有 【来源】 标签
- 更新格式或样式
- 批量添加前缀或后缀
